Clean, modular, & scalable code

        Angular directives and Bootstrap classes make HTML capable of some very powerful & expressive logic that I take advantage of every day in my development process. With Angular, I can keep a lot of the DOM logic inside of the HTML as opposed to hidden in the controllers somewhere, which results in code that is quick & easy to debug and understand. I also follow Google's HTML style guide for purposes of effiency and to be consistent with Google's search engine algorithms. I use useful & descriptive tag names for my elements instead of just using 'div' for everything as well as commenting parts of my code that may be of interest.

        Writing JavaScript is easily the largest part of my development process and probably the most important. I put a great deal of effort into finding an efficient, succinct, and easily understandable way of expressing my code. I also cut every bite of text and millisecond of loading time I can so users can have as smooth and coherent of an experience as possible. I compile & minify all of my code before launch for efficiency. I also follow Google's JavaScript style guide and John Papa's Angular style guide and comment all of my code so that it can be clean & readable to other developers. I research potential security risks and code around them, such as disallowing special characters in forms to avoid script injection.

Tools I use
Frameworks

        I use Angular.js and JQuery as my javascript frameworks of choice, although I mostly use jQuery in the form of jqLite, which is compact version of it built into Angular. I started using jQuery soon after I began learning web development and a couple months after that I began using Angular to replace much of the functionality of jQuery in my code. Angular's data binding, services, directives, and single-page application capabilities gave me a lot more power as a new developer than I otherwise would have with jQuery or plain JS.

        I also use Bootstrap for its easy table layouts and occasionally for its built-in classes, but I usually tend to write my own CSS classes in lieu of the ones it offers.

Development Tools

        With its popularity and synchronization with Github, as well as fact that I develop on GNU/Linux, Git was an easy choice as my version control system. Yeoman was one of the tools that taught me the most about web development because it sets up the basic structure of the app directory and many of the files that are needed to make the app, including files that new developers might not even know about. Yeoman also introduced me to Grunt and Bower, which are very powerful time and bite savers. Grunt's workflow automation greatly sped up my development time and enabled me to use a bunch of useful Grunt tasks like autoprefixer and JSHint as well as preprocessors such as Sass and CoffeeScript with ease.

Design Tools

        I use Gimp for the majority of my image editing needs, but for my design workflow I also use Balsamiq. My workflow begins with brainstorming rough Balsamiq mockups until I get a basic idea of what the layout of the project will be. I then recreate it in Gimp and start playing around with the specifics of the design, such as the colors, text, shadows, etc. I then do a final recreation of the layout in the browser and make any minor adjustments needed. I like to do the majority of my designing outside of the browser so that the limitations of HTML/CSS don't dictate the outcome of the design too much.

        I use D3 for data visualization such as the circular Github graph in the section above. It's a great tool and does some tricks that are normally difficult to do with just CSS/JS.
